# DurabilityInfo 2.0 Initial Audit

This audit records the 1.2.0 baseline before production behavior changes.

## Project matrix

| Project | Minecraft | Java | Loader | API/plugin |
|---|---:|---:|---|---|
| Fabric 26.1.2 | 26.1.2 | 25 | Fabric Loader 0.18.5 | Fabric API 0.145.4+26.1.2; Loom 1.15-SNAPSHOT |
| Fabric 26.2 | 26.2 | 25 | Fabric Loader 0.19.3 | Fabric API 0.153.0+26.2; Loom 1.17.13 |
| Forge 26.1.2 | 26.1.2 | 25 | Forge 64.0.8 | ForgeGradle [7.0.17,8) |
| Forge 26.2 | 26.2 | 25 | Forge 65.0.3 | ForgeGradle [7.0.17,8) |
| NeoForge 26.1.2 | 26.1.2 | 25 | NeoForge 26.1.0.1-beta | ModDevGradle 2.0.141 |
| NeoForge 26.2 | 26.2 | 25 | NeoForge 26.2.0.0-beta | ModDevGradle 2.0.141 |

Each project is self-contained under `src/main/java` and `src/main/resources`. Fabric uses `ItemTooltipCallback`; Forge and NeoForge use their `ItemTooltipEvent`. All six inject the HUD at the return of `extractRenderState`. Minecraft 26.1.2 targets `Gui` and checks `options.hideGui`; 26.2 targets `Hud` and checks `client.gui.hud.isHidden()`.

Configuration is `config/durabilityinfo.json`. Every loader has its own `ModConfig`, resolved through Fabric Loader, FMLPaths, or NeoForge FMLPaths. `load()` reads synchronously and silently swallows IO failures. Tooltip construction and HUD extraction both call it. The HUD constructs a list, slot records, strings, and text each frame. Low-durability state and item-break sounds are coupled to HUD rendering through a per-HUD map and one global cooldown.

Fabric registers a Cloth Config screen through optional Mod Menu integration, but compiles against unpinned `latest.release` versions. Forge and NeoForge register their built-in Mods-screen Config extension, whose screen exposes only five booleans. Mixins consist solely of `DurabilityInfoHUDMixin`.

## Baseline behavior

- Damageable tooltips can show remaining/max numbers, integer percentage, and a 20-character text bar.
- Unbreakable items are hidden unless enabled.
- The HUD renders armor, main hand, and offhand vertically as a bar or percentage.
- Four corner anchors and X/Y offsets are available.
- A single threshold flashes red and plays the item-break sound.
- Remaining and damage-taken modes exist, although damage-taken affects HUD percentage text only.
- Color boundaries are independently hard-coded in tooltip and HUD code.

## Duplication and risks

The 26.1.2 HUD implementations are byte-identical across loaders, as are the 26.2 implementations. Screens are duplicated by Minecraft version, Cloth screens are duplicated, and entrypoints/configs differ only around loader APIs. There is no shared testable durability/config/visibility/alert model. The 2.0 architecture therefore uses one loader-neutral shared source set compiled by every project, with narrow Minecraft and loader adapters retained per version.
